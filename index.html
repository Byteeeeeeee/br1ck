How It Works

The exploit works because of an issue CR50 chromebooks have when enrolling. If you reset it at just the right time, it will (some) corrupt data in the FWMP TPM space, resulting in a bricked FWMP. This allows us to boot into shimmer with CTRL + U and clear FWMP for when we fix the brick.

How I Found It

While trying to do CRSH2TTY, I ended up bricking my device multiple times (a total of four). One time, I randomly tried to disable write protection while unbricking with a SuzyQ, and surprisingly, it worked!
All of my exploits have been made due to CRSH2TTY in some way.

How to Do It

1. Go to the section below this one ("Find Reset Time"), memorize the time it gives you, then continue
2. Powerwash and go through the regular setup until the "Getting device ready" screen.
3. Get a stopwatch ready and wait for the "Enterprise Enrollment" screen. 4. Wait until you're at the time the file uploader gave you, perform an EC-Reset by holding Refresh and then clicking Power.
5. Reboot (Refresh + Power). If you're still in recovery mode, click TAB. If the recovery reason is not "Recovery Button Pressed", continue! Otherwise, retry until you get it.

How to use this?

1. Go to chrome://network#logs
2. Under the options section check all of the boxes.
3. Place the file in combined-logs/var/log/chrome/ into the file uploader
4. Click submit and it should show an alert with the reset times
